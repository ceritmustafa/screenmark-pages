# ScreenMark App Store Marketing Kit

> Last updated: 2026-03-25
> Scope: Mac App Store screenshots, app preview, poster frame, capture guidance

## Goal

ScreenMark is easiest to understand when people see motion and screen context.
The product page should therefore sell one clear idea:

**ScreenMark helps Mac presenters direct attention, annotate live, freeze a moment, and keep sessions moving without leaving the flow.**

The first impression should not focus on onboarding, pricing, or settings.
It should focus on the active presentation workflow.

## Recommended Launch Set

- 6 screenshots total
- 1 app preview video
- 1 poster frame derived from the strongest video moment
- 1 dark-mode screenshot inside the set

## Current Apple Asset Targets

Verified against Apple Developer docs on 2026-03-25:

- macOS screenshots: `16:10`
- Preferred masters: `2880 x 1800`
- Other accepted screenshot sizes include `2560 x 1600`, `1440 x 900`, `1280 x 800`
- macOS app preview: `1920 x 1080`
- App preview duration: `15-30 seconds`

## Visual Direction

Use the product's existing visual language from the app:

- Primary cool accent: blue
- Secondary warm accent: orange
- Clean rounded panels and floating overlays
- Light, airy marketing backgrounds with one dark hero

Do not build noisy, poster-like compositions that bury the real app.
The app capture must remain the hero.

- Primary feature: freeze frame + export

  - keep PNG/PDF/TIFF visible if the composition allows it

### Screenshot 4

- Supporting line: `Move from live content to a clean sketch space in one step.`
- Primary feature: whiteboard or blackboard mode
- Theme: light background

- Working title: `Keep sessions on time.`
- Supporting line: `Use a visible break timer without breaking the room's momentum.`

## Screenshot Layout Blueprint

Use these layout rules when building the final App Store screenshot masters in Figma.
They are intentionally simple so the app capture remains the hero.

### Shared Layout Rules

- Canvas size: `2880 x 1800`
- Safe content margin: `160 px`
- Headline block width: `1040-1120 px`
- Headline alignment: left
- App capture block: right side for screenshots `1-3`
- Capture block width target: `1240-1360 px`
- Badge style:
  - one compact pill
  - placed near the lower-left content area
  - no more than one badge per screenshot
- Background treatment:
  - soft gradient or soft color wash only
  - never let the background compete with the app window

### Screenshot 1 Layout

- Left column:
  - eyebrow: `MARKIT FOR MAC`
  - headline
  - one supporting line
  - one badge: `LIVE ZOOM`
- Right column:
  - large dark-mode app window
  - visible floating toolbar near top-center
  - zoomed focus region visible immediately
- Balance target:
  - left text column about `40%`
  - right app window about `60%`

### Screenshot 2 Layout

- Left column:
  - headline
  - supporting line
  - optional one-line explanation chip
- Right column:
  - light app window
  - annotation scene with box + arrow + note
  - avoid more than three visible annotation marks
- Balance target:
  - text `38-40%`
  - app capture `60-62%`

### Screenshot 3 Layout

- Left column:
  - headline
  - supporting line
  - badge: `EXPORT`
- Right column:
  - split visual story inside the app window
  - frozen content on one side
  - export outcome or export formats on the other
- Balance target:
  - keep export UI secondary to the actual marked-up screen

### Screenshot 4 Layout

- Top-left:
  - headline
  - supporting line
- Main area:
  - large whiteboard or blackboard scene
  - one clean sketch or concept path
- Balance target:
  - much more spacious than screenshots `1-3`
  - minimal chrome

### Screenshot 5 Layout

- Left:
  - headline
  - supporting line
- Right:
  - timer overlay visible on top of realistic content
  - timer should be legible at thumbnail size
- Balance target:
  - timer acts as proof of utility, not the entire composition

### Screenshot 6 Layout

- Left:
  - headline
  - supporting line
- Right:
  - menu bar or settings/control center
  - show one or two meaningful controls only
- Balance target:
  - avoid a dense preferences screenshot with tiny unreadable text

## App Preview Structure

Target length: `20-25 seconds`

The preview must work muted.
Every scene should be understandable through motion and a very short caption.

### Scene 1: Hook

- Duration: `0-4s`
- Caption: `Present with clarity.`
- Action:
  - show a live screen
  - zoom into the important region
  - show cursor focus
- Purpose:
  - immediate product comprehension

### Scene 2: Annotate

- Duration: `4-10s`
- Caption: `Draw on any screen.`
- Action:
  - draw one box
  - add one arrow
  - type one very short note
- Purpose:
  - show the product is interactive, not passive

### Scene 3: Freeze + Export

- Duration: `10-17s`
- Caption: `Freeze and save the moment.`
- Action:
  - freeze the screen
  - annotate once
  - hint at export or resulting output
- Purpose:
  - connect presentation help with practical follow-up

### Scene 4: Whiteboard or Timer Close

- Duration: `17-25s`
- Caption: `Keep the session moving.`
- Action:
  - switch to whiteboard, or
  - show a visible break timer over content
- Purpose:
  - close on utility and control

## App Preview Shooting Script

This is the recommended first-pass recording script.
Use it to capture a single clean demo session, then trim in post.

### Recording Setup

- Resolution: `1920 x 1080`
- Duration target: `22 seconds`
- Audio: optional, but assume many viewers will watch muted
- Cursor: visible and intentional
- Desktop: clean wallpaper, no notifications, no unrelated windows

### Scene-by-Scene Script

#### Scene 1

- Time: `0.0-3.5s`
- Caption on video: `Present with clarity.`
- Capture:
  - start on a presentation slide or dashboard
  - move cursor to the key region
  - activate live zoom
- Performance note:
  - keep the cursor movement slow enough to read
  - avoid a dramatic zoom jump

#### Scene 2

- Time: `3.5-8.5s`
- Caption on video: `Draw on any screen.`
- Capture:
  - draw one rectangle
  - add one arrow
  - type one short note such as `Focus here`
- Performance note:
  - do not cycle through every tool
  - show one clear annotation flow

#### Scene 3

- Time: `8.5-14.5s`
- Caption on video: `Freeze and save the moment.`
- Capture:
  - freeze the current screen
  - keep one annotation visible
  - reveal the saved result or export affordance
- Performance note:
  - this beat should communicate that the output survives the live moment

#### Scene 4

- Time: `14.5-18.5s`
- Caption on video: `Switch to whiteboard instantly.`
- Capture:
  - transition into whiteboard mode
  - draw one simple sketch or concept path
- Performance note:
  - keep strokes minimal and readable

#### Scene 5

- Time: `18.5-22.0s`
- Caption on video: `Keep the session moving.`
- Capture:
  - show the visible timer over content or over the whiteboard scene
  - hold the final frame long enough to read
- Performance note:
  - end calmly
  - avoid ending on settings, paywall, or menu clutter

## Video Edit Notes

- Use very fast cuts only if the cursor path remains readable
- Prefer short dissolves or direct cuts over flashy motion graphics
- Keep captions large and short
- Do not overlay long paragraphs
- Poster frame should come from scene 1 unless scene 3 produces a stronger still

## Demo Content Plan

To make production easier, prepare the demo materials before recording:

### Demo Surface A: Presentation

- one headline slide
- one chart or three-column comparison
- one highlighted takeaway area

### Demo Surface B: Document

- one staged spec or notes page
- a heading
- three short text blocks

### Demo Surface C: Whiteboard

- one simple flow or concept sketch
- no more than 5-7 visible strokes

## Quick Build Order For Design

If you are rebuilding this manually in Figma without MCP:

1. Create three `2880 x 1800` masters for screenshots `1-3`.
2. Duplicate one master for screenshots `4-6`.
3. Create four or five `1920 x 1080` storyboard frames.
4. Paste the English copy first.
5. Place real app captures.
6. Tune spacing only after all captures are in place.

## Poster Frame Recommendation

Use a frame derived from screenshot 1 or video scene 1.

Best poster ingredients:

- zoomed content area
- visible cursor focus
- floating toolbar
- one short headline

Do not use:

- settings screens
- pricing UI
- onboarding
- empty whiteboard

## Capture Guidance

The safest capture strategy is to produce owned demo content designed for marketing use.

Use demo source material such as:

- a fake keynote-style deck
- a sample analytics dashboard
- a staged document
- a staged browser page

Avoid using copyrighted slides, client data, or recognizable third-party brands.

## Capture Checklist Per Scene

For every capture:

- use clean desktop content
- hide distracting notifications
- keep the dock/menu bar intentional
- use one main action per image
- keep annotation colors consistent with product branding
- avoid overly long headline text inside the capture itself

## Figma Composition Rules

When rebuilding these in Figma:

- screenshot masters: `2880 x 1800`
- preview storyboard masters: `1920 x 1080`
- keep headline area on the left and app capture on the right for the first 3 shots
- use one dark hero screenshot and otherwise lighter frames
- keep app capture large, roughly `45-55%` of the canvas width
- use blue and orange as accents, not as full-background noise

## Production Order

Recommended order of execution:

1. Create demo source content for the presentation, document, and whiteboard scenes.
2. Capture real ScreenMark screens in the app.
3. Place those captures into the screenshot masters.
4. Export screenshot set.
5. Record the 20-25 second preview video.
6. Pick the poster frame from scene 1.
